# 玩家系统设计大纲

## 一、玩家属性

| 属性 | 初始值 | 说明 |
|---|---|---|
| 名称 name | 玩家输入 | 玩家昵称 |
| 生命 hp | 50 | 初始 50 点，归零则游戏结束 |
| 金币 gold | 3 | 初始 3 枚 |
| 回合数 round | 1 | 当前回合数 |
| 棋子列表 pieces | 空 | 玩家拥有的棋子（含备战席和战场） |

---

## 二、经济系统（金币）

### 金币获取
- **每回合固定收入**：3 枚金币
- **回合递增奖励**：每经过一回合，固定收入 +1 枚
  - 第 1 回合：收入 3g
  - 第 2 回合：收入 4g
  - 第 3 回合：收入 5g
  - ...
  - **上限 10g**：单回合收入最多 10 枚（第 8 回合达到上限后不再增加）
- **战斗胜利奖励**：+1g
- **利息（可选）**：每 5g 存款产生 1g 利息，上限 5g（鼓励攒钱策略）

### 金币用途
- 购买棋子（按棋子 gold 值扣减）
- 升级棋子（按等级倍数扣减）
- 升级商店（花费 = 当前商店等级 + 4）
- 刷新商店（1g/次）
- 合并进阶职业（免费，但消耗两个棋子）

### 金币计算公式
```
每回合收入 = min(3 + (当前回合数 - 1), 10)
战斗胜利额外 +1g
```

## 三、商店系统

### 商店属性
| 属性 | 初始值 | 说明 |
|---|---|---|
| 等级 level | 1 | 商店等级，影响可刷新棋子的等级上限和展示数量 |
| 展示数量 | 等级 + 2 | 每次刷新展示的可购买棋子数（1 级展示 3 个，5 级展示 7 个） |
| 最高等级 | 5 | 商店等级上限 |

### 商店升级
- 花费金币 = 当前商店等级 + 4
  - 1→2 级：5g
  - 2→3 级：6g
  - 3→4 级：7g
  - 4→5 级：8g
- 商店等级越高，可刷出更高等级的棋子，展示数量越多

### 刷新规则
- 刷新花费 1g
- 刷新后展示新的棋子列表（数量 = 商店等级 + 2）
- **不刷新进阶职业**：进阶职业只能通过合并获得，商店不会直接出售
- **等级限制**：只刷新棋子等级 ≤ 商店等级的基础职业
  - 商店 1 级：只刷出 1 级棋子
  - 商店 3 级：可刷出 1-3 级棋子
  - 商店 5 级：可刷出 1-5 级棋子

### 刷新概率（按棋子等级）
商店等级越高，高等级棋子出现概率越高，但仍低于低等级棋子。

| 棋子等级 | 商店1级 | 商店2级 | 商店3级 | 商店4级 | 商店5级 |
|---|---|---|---|---|---|
| 1 级 | 100% | 70% | 50% | 35% | 25% |
| 2 级 | - | 30% | 30% | 25% | 20% |
| 3 级 | - | - | 20% | 20% | 20% |
| 4 级 | - | - | - | 20% | 20% |
| 5 级 | - | - | - | - | 15% |

> 每行概率为该商店等级下刷出对应等级棋子的概率，同一等级内 7 个基础职业随机出现。

### 购买规则
- 玩家选择展示列表中的棋子，支付对应 gold 值购买
- 购买后该位置置空，下次刷新补充
- 购买的棋子进入玩家备战席

## 四、生命系统

### 初始生命
50 点

### 战斗失败扣血
战斗失败时，扣血量 = **敌方所有存活棋子的等级之和**

- 例：敌方剩 3 个存活棋子，等级分别是 2、3、1，则扣 6 点生命
- 敌方全军覆没则不扣血（即玩家胜利）

### 战斗胜利
- 不扣血
- 获得 1g 胜利奖励

### 游戏结束
- 玩家生命 ≤ 0 → 游戏失败
- 击败所有 AI 对手 → 游戏胜利

## 五、回合流程

```
回合开始
  │
  ▼
1. 发放金币
   收入 = min(3 + (回合数 - 1), 10)
   若上回合战斗胜利，额外 +1g
  │
  ▼
2. 准备阶段（玩家操作）
   - 查看商店，购买棋子
   - 升级现有棋子
   - 升级商店（花费 = 当前商店等级 + 4）
   - 排列阵型（设置棋子站位 0-6）
   - 合并同等级同系别棋子为进阶职业
   - 刷新商店（可选，1g/次）
  │
  ▼
3. 战斗阶段
   - 玩家棋子 vs AI 棋子，酒馆战棋式自动战斗
   - 按从左到右顺序依次行动
  │
  ▼
4. 结算阶段
   - 若玩家败：扣血 = 敌方存活棋子等级之和
   - 若玩家胜：+1g 奖励
   - 回合数 +1
  │
  ▼
5. 判定游戏是否结束
   - 玩家生命 ≤ 0 → 失败
   - 击败所有 AI → 胜利
   - 否则 → 进入下一回合
```

## 六、AI 玩家（AIPlayer，继承 Player）

AI 玩家与玩家共享相同的属性和经济规则，操作由 AI 用**贪心算法**自动决策：每一步选择当前收益最大的操作，不做全局搜索。

### 贪心决策流程

每回合准备阶段，AI 按以下优先级依次执行（每步都选当前最优）：

```
回合开始
  │
  ▼
1. 合并检查（收益最大，免费）
   遍历备战席所有棋子对，若存在可合并的（同等级同系别不同职业），
   立即合并，循环直到无可合并组合
  │
  ▼
2. 升级商店（长期收益）
   若金币 ≥ 商店升级费用 + 2（留余量），且商店未满级，则升级商店
  │
  ▼
3. 购买棋子（贪心选性价比最高的）
   计算商店中每个棋子的性价比 = getPower() / gold
   若金币足够且备战席未满（< 7 个），购买性价比最高的
   重复直到买不起或备战席满
  │
  ▼
4. 升级棋子（贪心选战力最低的优先升）
   若金币 ≥ 某棋子升级费用，选备战席中 getPower() 最低的升级
   （提升空间最大的优先）
  │
  ▼
5. 刷新商店（若还有富余金币且备战席未满）
   若金币 ≥ 3 且备战席有空位，刷新商店后回到步骤 3
   最多刷新 2 次（防止无限刷新）
  │
  ▼
6. 排列阵型（贪心摆位）
   - 嘲讽棋子放最左（位置 0-2）
   - 潜行棋子放右侧（位置 4-6）
   - 其他棋子按 getPower() 从高到低放中间
  │
  ▼
7. 补满战场
   若战场有空位且备战席有棋子，按战力从高到低依次上场
```

### 贪心策略说明

| 决策点 | 贪心标准 | 理由 |
|---|---|---|
| 合并 | 有就合 | 免费提升，收益最大 |
| 升级商店 | 金币富裕时升 | 提高后续棋子质量上限 |
| 购买 | 性价比 = 战力/金币 | 单位金币收益最大化 |
| 升级棋子 | 战力最低的先升 | 提升空间最大 |
| 刷新 | 备战席有空位才刷 | 避免浪费金币 |
| 摆位 | 嘲讽左/潜行右/输出中 | 保护脆皮，吸收伤害 |

### 贪心的局限性
- 不考虑多步组合（如"这回合攒钱下回合买更好的"）
- 不预判对手阵型
- 但实现简单、运行快、效果可接受，适合课程设计

## 七、类设计

### Player 类
```
属性：
- name: string              玩家名
- hp: int                   生命值（初始 50）
- gold: int                 金币（初始 3）
- round: int                回合数（初始 1）
- pieces: vector<ChessPiece*>  棋子列表（备战席）
- board: array<ChessPiece*, 7>  战场阵型（7 个位置）
- shopLevel: int            商店等级（初始 1，最高 5）
- shop: vector<ChessPiece*> 当前商店展示的棋子
- isAI: bool                是否为 AI

方法：
- gainGold()               发放本回合金币
- buyPiece(index)          购买商店中指定位置的棋子
- upgradePiece(piece)      升级棋子
- upgradeShop()            升级商店（花费 = 商店等级 + 4）
- refreshShop()            刷新商店（1g）
- mergePiece(a, b)         合并进阶职业
- arrangeBoard()           排列阵型
- takeDamage(amount)       战斗失败扣血
- isDead()                 判断是否死亡
- display()                显示玩家状态
```

### AIPlayer 类（继承 Player）
```
重写方法：
- autoPlay()              AI 自动操作（购买/升级/排列/合并/升级商店）
```
